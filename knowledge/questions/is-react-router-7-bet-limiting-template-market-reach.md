## Context

Our competitive analysis states that "85%+ of templates use App Router" (Next.js) and that "Drizzle is gaining share vs Prisma." Our flagship template and all active template variants use React Router 7 + Hono + Drizzle — a stack that's technically excellent but represents a minority framework choice. The GTM strategy positions this as differentiation ("the only React Router 7 + Hono template"), but differentiation only works if customers see it as an advantage rather than a barrier.

The SaaS template market is a $149-$1,199 purchase decision. Buyers want to ship fast, and framework familiarity is a major factor. If 85% of the market is comfortable with Next.js, we're fishing in the remaining 15% pond.

## What We Know

- 85%+ of competing SaaS templates use Next.js App Router
- ShipFast ($199, Next.js) is the market leader by volume
- Makerkit ($349/yr) and Supastarter ($299+) both offer Next.js as primary
- We have an older Next.js template (launchpad-saas-template) using Next.js 14 + Prisma, but it appears inactive
- React Router 7 has strong technical merits (SSR, RSC support coming, Remix heritage)
- Our GTM positions React Router 7 as a differentiator, not a limitation
- The revenue roadmap projects $1.5K-$9K/mo from template sales

## What We Don't Know

- What percentage of SaaS template buyers specifically want React Router 7 vs Next.js?
- Are there developers who would buy our template but won't because it's not Next.js?
- Is the React Router 7 ecosystem mature enough for production SaaS? (middleware, deployment guides, hosting support)
- Would maintaining a Next.js variant alongside React Router 7 be feasible with AO's velocity?
- What's the actual search volume for "React Router SaaS template" vs "Next.js SaaS template"?
- Is Remix/React Router 7 growing or plateauing in adoption?

## Suggested Investigation

1. **Search volume analysis**: Compare Google Trends and keyword data for "React Router SaaS template" vs "Next.js SaaS template" vs "Remix SaaS template"
2. **Competitor sales data**: Check if ShipFast or Makerkit publish any revenue numbers that indicate market size for Next.js templates
3. **Community sentiment**: Survey r/reactjs, r/typescript, and React Router Discord for framework preference in production SaaS
4. **Dual-template feasibility**: Estimate the effort to maintain a Next.js variant — AO could potentially maintain both if the architecture is similar enough
5. **Conversion tracking**: Once the template is for sale, track how many visitors bounce when they see "React Router 7" vs convert
