# Per-Repo Knowledge Base

> Each file in this directory documents a single repository in the launchapp-dev org.

## File Naming

Files are named after their repo: `<repo-name>.md` (e.g., `ao-cli.md`, `launchpad-baas.md`).

## Expected Structure Per File

Each repo file should contain:

- **Purpose** — What this repo does and who it's for
- **Tech Stack** — Languages, frameworks, key dependencies
- **Architecture** — How the code is organized, key directories
- **Dependencies** — What org repos it depends on / what depends on it
- **API Surface** — Key exports, endpoints, or interfaces
- **Current Status** — Active development, stable, maintenance, or archived
- **Recent Activity** — Latest PRs, issues, releases
- **Known Issues** — Open bugs, tech debt, blockers
- **Team** — Who maintains it (if known)
