# Pending Decisions

Decision packages awaiting founder review. Generated weekly by the decision-packager agent.

## How It Works

1. The decision-packager reads all `proposed` action items
2. It creates a batch file here with approve/reject/defer for each
3. Founders (Sami or Rafael) edit the file and mark their decisions
4. The decision-processor reads the decisions and updates action statuses

## File Format

Each batch is named `<date>-batch.md` with a table of actions to decide on.
