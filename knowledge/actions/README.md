# Action Items

Concrete action items extracted from answered questions. Generated by the action-extractor agent.

## Lifecycle

1. **proposed** — action extracted, needs review
2. **approved** — reviewed and approved for execution
3. **in-progress** — work has started
4. **done** — action completed
5. **rejected** — decided not to pursue

## File Format

Each action is its own .md file with YAML frontmatter. Named with kebab-case describing the action (e.g., `sunset-launchpad-baas-sdks.md`).

### Template

```markdown
---
title: "Sunset LaunchPad BaaS legacy SDKs"
priority: high
status: proposed
effort: medium
category: product-strategy
source_question: knowledge/questions/should-we-sunset-launchpad-baas.md
owner: unassigned
target_repos:
  - launchpad-identity-sdk
  - launchpad-payments-sdk
  - launchpad-storage-sdk
generated_by: action-extractor
generated_at: 2026-03-19
---

## Context

Why this action matters — link back to the answered question.

## Scope

What specifically needs to happen, step by step.

## Dependencies

What must happen first before this can start.

## Success Criteria

How we know this is done.

## Notes

Any additional context, risks, or considerations.
```

### Frontmatter Fields

| Field | Type | Required | Values |
|-------|------|----------|--------|
| `title` | string | yes | Clear imperative action |
| `priority` | enum | yes | `critical`, `high`, `medium`, `low` |
| `status` | enum | yes | `proposed`, `approved`, `in-progress`, `done`, `rejected` |
| `effort` | enum | yes | `small` (days), `medium` (weeks), `large` (months) |
| `category` | enum | yes | `product-strategy`, `technical-debt`, `competitive`, `revenue`, `architecture`, `operations`, `market-trends`, `security` |
| `source_question` | string | yes | Path to the question file that prompted this |
| `owner` | string | yes | Who owns this (`unassigned` if no one yet) |
| `target_repos` | list | no | Repos this action affects |
| `generated_by` | string | yes | Agent that created it |
| `generated_at` | date | yes | ISO date |
| `completed_at` | date | no | When completed |
