# Architecture Diagrams

Mermaid architectural diagrams for every product. Generated by the architecture-diagrammer agent from reading actual source code.

## Diagram Types

| Type | Purpose |
|------|---------|
| `system` | High-level system architecture — services, databases, APIs, external dependencies |
| `data-flow` | How data moves through the system — requests, events, queues |
| `dependency` | Package/module dependency graph — internal and external deps |
| `deployment` | How the product is deployed — containers, cloud services, CI/CD |

## File Format

Each diagram is its own .md file with YAML frontmatter and Mermaid code blocks.

### Template

```markdown
---
title: "AO CLI — System Architecture"
product: ao-cli
type: system
status: current
source_repos:
  - ao-cli
generated_by: architecture-diagrammer
generated_at: 2026-03-19
last_verified: 2026-03-19
---

## Overview

Brief description of what this diagram shows.

## Diagram

\```mermaid
graph TD
    A[Component] --> B[Component]
\```

## Notes

Key architectural decisions, trade-offs, or things to watch.
```

### Frontmatter Fields

| Field | Type | Required | Values |
|-------|------|----------|--------|
| `title` | string | yes | Product name + diagram type |
| `product` | string | yes | Product slug (e.g., `ao-cli`, `saas-template`) |
| `type` | enum | yes | `system`, `data-flow`, `dependency`, `deployment` |
| `status` | enum | yes | `current`, `outdated`, `draft` |
| `source_repos` | list | yes | Repos this diagram covers |
| `generated_by` | string | yes | Agent that created it |
| `generated_at` | date | yes | ISO date |
| `last_verified` | date | yes | Last time the diagram was verified against source code |

## Naming

Files named: `<product>-<type>.md` (e.g., `ao-cli-system.md`, `saas-template-data-flow.md`)
